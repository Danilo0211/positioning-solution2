A JavaScript data attributes example by Danilo Araujo

This webpage demonstrates how to dynamically style elements using JavaScript to filter a collection of creature images. By utilizing custom data attributes in the HTML, each creature is categorized by its type and habitat. When a user clicks the interactive buttons, a JavaScript file uses document.querySelectorAll() to grab these specific groups as NodeLists. The script then loops through these elements to instantly update their background colors and borders based on the selected category.

Daily Grind project in JavaScript

This single-page application displays dynamic, daily rotating content by using the JavaScript Date object to automatically detect the current day of the week. Based on the active day, conditional logic (like a switch block) automatically updates the page's color scheme, featured image, all tags, and descriptive text. Additionally, the script utilizes URL query string parameters, allowing users to override the automatic system clock and manually cycle through any day's special using interactive navigation links.

Live Demo GitHub Repo

Season Stylin' with CSS & JavaScript

This webpage dynamically transforms the user interface for a seasonal retail brand using JavaScript data objects to manage content across all four seasons. When a user clicks a season button, a script swaps out the page's color scheme, logo image, featured clothing item, and promotional text to reflect that time of year.The page also features a "Default" reset capability and includes a brief overview explaining how JavaScript manipulates the DOM to achieve these real-time style transitions.

PHP server-side application using JavaScript

This web application utilizes asynchronous JavaScript (AJAX/Fetch) to dynamically request and display random quotes from a server-side PHP backend. The implementation progresses from a manual, click-triggered quote retrieval system to an automated process that fetches new content both on initial page load and at set timed intervals. To ensure seamless communication between the frontend client and the remote PHP API across different domains, the server is configured to handle Cross-Origin Resource Sharing (CORS) permissions.

Bill's Advice: Take time to explain clearly and perfectly what's going on, how things work, what you've learned, and try to do so succinctly. When employers see what we've internalized, they imagine we can act as a valuable resource to them. We are selling our skills as a resource and asset, and the better job we do articulating this, the more easily an employer can see our value.

It may be MORE important to be able to clearly explain things over actually knowing them! At least if I can explain myself, I'm showing I can adapt in a dynamic manner, and if I know things but can't explain myself, no one will know what I can do.
